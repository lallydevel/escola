1. Estrutura de Pastas Sugerida
Recomendo separar o projeto em dois diretórios principais no repositório para isolar as responsabilidades
.
/sistema-chamada
├── frontend/             # React + Vite + TS
│   ├── src/
│   │   ├── components/   # Componentes reutilizáveis (Botões, Inputs) [3, 4]
│   │   ├── hooks/        # Custom hooks e lógica de React Query [4, 5]
│   │   ├── pages/        # Telas (Home, ListaAlunos, Chamada) [6]
│   │   ├── services/     # Chamadas de API [4]
│   │   └── types/        # Interfaces TypeScript [7]
|   |   |_  assets/       # img, css e js
├── backend/              # Node.js + TypeScript
│   ├── src/
│   │   ├── routes/       # Definição dos Endpoints
│   │   ├── controllers/  # Lógica de negócio
│   │   └── models/       # Estrutura de dados
└── .gitignore            # Essencial para ignorar node_modules e arquivos sensíveis [8, 9]


2. Definição do Contrato de Dados (Interfaces)
Para que seu grupo trabalhe em sintonia, definiremos as Interfaces primeiro. Elas garantem que o frontend saiba exatamente o que o backend enviará
.
// frontend/src/types/index.ts ou compartilhado
export interface IAluno {
  id: string;
  nome: string;
  matricula: string;
}

export interface IRegistroChamada {
  alunoId: string;
  data: string; // ISO string
  presente: boolean;
}
